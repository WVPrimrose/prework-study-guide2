    Terms and Commands

  • branch: a copy or version of the code, so that it can be worked on independently to prevent overwriting each others' work
    • Main (or default branch): the starting branch that will hold the final product of the project
      • NO CODING DIRECTLY ON THE MAIN BRANCH!!!!!
    • Feature (or working branch): a branch that adds any new features being currently developed but not merged
      into the main branch
    • Developed branch: hold code for multiple feautre branches but not fully merged into the main branch

    NOTE: Any changes to any feature branches that have NOT been merged into the main branch do NOT effect the final product
    UNTIL they're approved, tested, and merged, so any new features don't disrupt the main code.

  • That's why we have pull requests (PRs): other developers can comment and/or request any changes affecting the final product.

      • Commands

  • git status: shows what branch we are currently on
  • git checkout -b [branch-name]: creates a new branch and switches over to it
  • git add: adds any file to the next commit stage
  • git merge [branch]:merging a branch to a current one

    • NOTE: doesn't necessarily have to be the main branch

  • git commit -m "[descriptive message]": adding what we did to the code through descriptive messages, so we can communicate to
    our team members.

    • NOTE: Please commit often, especially because communication can be difficult through the Internet,
    especially through different timezones.

      • CCS

      stands for Cascading Style Sheets provides how any webpage content should be presented through rules for specific HTML elements.

        Examples

      • colors
      • aesthetics
      • visual layouts
      • organization

        Different Ways

      1. Inline
        2. Styling CCS rules directly into the HTML file's elements. Problem is you have to style each and every individual HTML tag.
      2. Internal
        3. Styling CCS rules directly into the HTML file's head. If you have multiple styling elements, this makes changes harder to navigate
      3. External
        4. Stying CCS rules in a separate file that'll be linked to the HTML file's head. This is the prefered method because this makes changes
        much easier in an organizaed fashion.
